01 / Overview

Surveillance that works when everything else doesn't

SAHA-Care is a proposed offline-first, community-based surveillance (CBS) mobile app for infectious disease detection in conflict-affected environments. Its initial deployment context is the Gaza Strip, where ongoing conflict has collapsed existing health infrastructure and displaced millions.

Community health workers and displaced individuals can report standardized case definitions even during connectivity blackouts — falling back to structured SMS/USSD when mobile data is unavailable. Reports aggregate to a GCP cloud backend with automated triage.

02 / The Problem

Why existing systems fail in crisis

Infrastructure
Surveillance systems go dark precisely when disease risk is highest
Displacement
Hard-to-reach populations remain invisible to traditional reporting chains
Connectivity
Internet-dependent tools fail during blackouts — the norm in active conflict
Fragmentation
No shared case definitions across humanitarian actors, leading to data silos
Trust
Top-down systems exclude community knowledge and erode local agency

Design Rationale

Every layer was chosen around one constraint: the system must work when infrastructure is degraded or absent. Offline-first on the device, SMS fallback for zero-data environments, serverless on the backend to minimize cost and maintenance overhead.

Android · Kotlin
Built for the field

Android dominates device ownership in LMICs. Native Kotlin gives direct access to WorkManager for guaranteed background sync and the Android Keystore for encryption — critical APIs that cross-platform frameworks abstract poorly.

Africa's Talking
SMS/USSD when data is unavailable

Chosen over Twilio for direct MENA carrier integrations and native USSD support. USSD works on any SIM with zero data, extending reporting to feature phones and fully offline conditions.

FastAPI · Cloud Run
Scalable, cost-effective backend

FastAPI provides fast development, async handling, and automatic validation. Cloud Run scales to zero when idle — eliminating hosting costs between uses — and deploys directly from GitHub Actions on every push.

Firestore · Firebase Auth
Real-time data with enforced access control

Firestore's document model suits self-contained case reports and its real-time listeners power live dashboards. Firebase Auth enables phone-number OTP login for field workers, with security rules enforcing data isolation at the database level.
